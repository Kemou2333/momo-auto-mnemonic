## 已确定的硬约束（来自 API 和工程层面）

- 每个词 1~3 条例句（墨墨 App 最多展示 3 条）
- 每条例句由两部分组成：英文整句 + 中文整句翻译
- 多义词尽量每条例句对应一个不同义项，单义词 1 条够了，不要硬凑
- 不要写义项重复的例句
- `phrase_en` 必须自然包含该词的某种形态（原形、复数、过去式、派生形式都行）
- 文本字段必须用 Python 三引号 `"""..."""` 包裹（避免引号冲突）

## 参考：墨墨官方例句的形态（来自 App 截图）

英文句子里目标词高亮（绿色），中文翻译里对应中文短语带虚线下划线。这部分是
**墨墨 App 自动渲染**的，我们写例句时**不需要**手动标注高亮位置——API 也不接受
这个字段。我们只需要给出干净的英文句子和干净的中文翻译。

示例（familiar，墨墨官方）：

```
英文：The park looked strangely familiar, though he knew he'd never been there before.
中文：尽管他知道自己以前没来过这里，但这个公园看上去却似曾相识。
```

```
英文：Her name is familiar, but I can't remind where I've heard it.
中文：她的名字很耳熟，但我记不起在哪儿听过。
```

注意：这两条例句对应的是 familiar 的两个不同义项（"似曾相识" vs "耳熟"），是
"多义词覆盖不同义项"的范例。
